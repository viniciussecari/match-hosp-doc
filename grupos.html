Grupos e Quartos

Gerenciamento de grupos formados e funcionalidade de desfazer match

Visao Geral

Quando dois participantes aceitam mutuamente um convite, um grupo e criado automaticamente. O grupo representa a alocacao do quarto e pode ser travado (locked) pela agencia.

Status do Grupo

MATCHED Grupo formado, ativo
LOCKED Travado, nao pode desfazer

Capacidade (MVP)

  • - Padrao: 2 pessoas por quarto
  • - Configuravel por evento
  • - Grupo fecha quando atinge capacidade

Endpoints

GET /guest/quarto

Retorna informacoes do grupo/quarto do pax logado

// Response 200 (com grupo)
{
  "grupoId": 789,
  "status": "MATCHED",
  "membros": [
    { "paxId": 100, "nome": "Joao Silva", "isMe": true },
    { "paxId": 123, "nome": "Maria Santos", "isMe": false }
  ],
  "podeDesfazer": true,
  "desfazerAte": "2026-02-15T23:59:59Z"
}

// Response 200 (sem grupo)
{
  "grupoId": null,
  "status": null,
  "membros": [],
  "podeDesfazer": false
}
POST /guest/match/desfazer

Desfaz o match atual (sai do grupo)

// Request
{ "motivo": "Prefiro ficar com outra pessoa" }

// Response 200
{
  "success": true,
  "message": "Match desfeito com sucesso"
}

// Response 409 (Motor bloqueou)
{
  "code": "LOCKED",
  "message": "O grupo esta travado e nao pode ser desfeito"
}
POST /admin/grupos/{id}/lock

Trava o grupo (Admin)

POST /admin/grupos/{id}/unlock

Destrava o grupo (Admin)

1) Tabela match_grupo_quarto
2) Criacao de Grupo (ao aceitar convite)
3) Desfazer Match (Backend)
4) Lock/Unlock (Admin)
5) Lock Automatico
6) Frontend - Tela do Quarto
7) Regras de Desfazer (Config)
